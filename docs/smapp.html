SMAPP API documentation version v1

http://smapp.sorters.io/api/{version}

  • version: required (v1)

Documentation for the SMAPP API

Authentication for the use of the API is by using the Authorization Header with the API Token provided on installation.

/products

/products get post

get

List all the available Products.

post

Create or Modify a Product. If a parameter 'id' for an existing Product is passed in the body the resource will update the existing Product.

get /products

List all the available Products.

  • Response

HTTP status code 200

A list of Products.

Body

Media type: application/json

Type: array of Product

Items: Product

  • id: required (integer)

    A unique identifier for the Product.

    Examples:

    27
  • name: required (string)

    A unique name for the Product.

    Examples:

    Keyboard
  • description: required (string)

    A description for the Product.

    Examples:

    Wireless standard QWERTY keyboard
  • category_id: (integer)

    The identifier of the Category to which the Product belongs, if any.

    Examples:

    3
  • created_at: required (datetime)

    The date when the Product was first created.

    Examples:

    2016-12-02T15:01:12Z
  • updated_at: required (datetime)

    The date when the Product was last modified.

    Examples:

    2016-12-06T12:33:52Z

HTTP status code 401

Unauthorized

post /products

Create or Modify a Product. If a parameter 'id' for an existing Product is passed in the body the resource will update the existing Product.

  • Response

Type: object

    Examples:

    Wireless standard QWERTY keyboard
  • category_id: (integer)

    (Optional) The identifier of the Category to which the Product belongs, if any.

    Examples:

    3

HTTP status code 200

The ID of the Product created or modified.
